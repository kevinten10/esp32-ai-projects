# AI 辅助开发指南

## 如何与 AI 协作开发 ESP32 项目

### 1. 项目初始化

**提示词示例**:
```
我要在 ESP32 上开发一个 [项目名称]，需要实现以下功能：
- 功能 1
- 功能 2

我使用的硬件：
- ESP32 开发板型号
- 传感器/模块列表

请帮我创建项目结构和初始化代码。
```

### 2. 代码生成

**提示词示例**:
```
请为 [模块名称] 编写驱动程序，要求：
- 使用 I2C/SPI 接口
- 引脚：SDA=21, SCL=22
- 实现 [具体功能]

请提供完整的 .h 和 .cpp 文件。
```

### 3. 调试协助

**提示词示例**:
```
我的代码出现以下错误：
[粘贴错误信息]

这是相关代码：
[粘贴代码]

请分析原因并提供解决方案。
```

### 4. 代码优化

**提示词示例**:
```
请优化这段 ESP32 代码：
- 降低功耗
- 提高响应速度
- 增加错误处理

[粘贴代码]
```

## 常用开发模式

### 模式 1: 增量开发
1. 让 AI 生成基础框架
2. 逐步添加功能模块
3. 每步都进行测试

### 模式 2: 组件复用
1. 创建通用组件库
2. 让 AI 适配到新项目
3. 维护组件文档

### 模式 3: 快速原型
1. 描述功能需求
2. AI 生成完整原型
3. 迭代优化

## 提示词模板

### 硬件配置
```
硬件配置：
- 开发板：[型号]
- 传感器：[名称 + 接口]
- 执行器：[名称 + 控制方式]
- 通信：[WiFi/Bluetooth/MQTT]
```

### 功能需求
```
功能需求：
1. [功能描述]
2. [输入/输出]
3. [性能要求]
```

### 代码风格
```
代码要求：
- 使用 Arduino 框架
- 添加详细注释
- 遵循 ESP-IDF 命名规范
- 包含错误处理
```

## 常见问题解决

| 问题类型 | AI 协助方式 |
|----------|-------------|
| 编译错误 | 粘贴错误日志，获取修复方案 |
| 硬件不工作 | 提供接线图，检查配置 |
| 性能问题 | 描述瓶颈，获取优化建议 |
| 内存不足 | 提供使用情况，获取优化方案 |

## 最佳实践

1. **明确具体**: 详细描述硬件和需求
2. **小步快跑**: 每次请求一个明确的功能
3. **代码审查**: 让 AI 解释关键代码段
4. **文档同步**: 让 AI 同步更新文档

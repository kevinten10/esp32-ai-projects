# 声音检测与拍手控制

基于 ESP32 + MAX9814 麦克风模块的声音控制系统，通过拍手次数控制家电设备。

---

## 功能特性

| 功能 | 说明 |
|------|------|
| 声音电平监测 | 实时采样麦克风音量，串口波形输出 |
| 拍手识别 | 识别 1~3 次拍手模式 |
| 设备控制 | 控制灯光（继电器1）和风扇（继电器2） |
| OLED 显示 | 显示音量电平条、波形、设备状态 |
| 蜂鸣器反馈 | 识别成功后蜂鸣提示 |
| 串口波形 | 配合 Arduino Serial Plotter 可视化音量 |

---

## 拍手命令说明

| 操作 | 动作 | 蜂鸣反馈 |
|------|------|---------|
| 拍手 1 次 | 切换灯光开/关 | 短鸣 1 次 |
| 拍手 2 次 | 切换风扇开/关 | 短鸣 2 次 |
| 拍手 3 次 | 全部关闭 | 短鸣 3 次 |

> 两次拍手间隔需 < 500ms 才算一组，超时后自动执行命令

## 引脚连接

```
ESP32            MAX9814 麦克风
GPIO 36 (VP) --> OUT（模拟输出）
3.3V         --> VCC
GND          --> GND
GND          --> GAIN（接地=40dB增益，推荐）

ESP32            继电器模块
GPIO 26      --> IN1（灯光）
GPIO 27      --> IN2（风扇）
3.3V/5V      --> VCC
GND          --> GND

ESP32            OLED SSD1306
GPIO 21      --> SDA
GPIO 22      --> SCL
3.3V         --> VCC
GND          --> GND

ESP32            蜂鸣器
GPIO 25      --> 正极（+）
GND          --> 负极（-）
```

> 💡 MAX9814 的 GAIN 引脚：
> - 接 GND = 40dB（拍手检测推荐）
> - 浮空 = 50dB
> - 接 VCC = 60dB（灵敏度最高，但更易误触发）

## 参数调节

编辑 `src/main.cpp` 中的宏定义：

```cpp
#define SAMPLE_WINDOW_MS  50    // 采样窗口时间，越大越稳定，响应越慢
#define CLAP_THRESHOLD    600   // 拍手检测阈值（0-4095）
#define CLAP_TIMEOUT_MS   500   // 等待下一次拍手的超时时间
#define MIN_CLAP_INTERVAL 100   // 两次拍手最小间隔（消抖）
```

### 阈值校准方法

1. 打开 Serial Plotter
2. 在安静环境中观察背景噪声电平（通常 50-200）
3. 拍一次手，观察峰值（通常 1000-3000）
4. 将 `CLAP_THRESHOLD` 设置为噪声电平的 3-5 倍

## 常见问题

**Q: 拍手没有反应？**  
A: 降低 `CLAP_THRESHOLD`（如改为 400）。打开 Serial Plotter 观察拍手时的电平峰值，设置阈值为峰值的 60%。

**Q: 误触发太频繁？**  
A: 提高 `CLAP_THRESHOLD`，或将 MAX9814 的 GAIN 引脚接地（降低增益到40dB）。

**Q: 拍两下只识别到一下？**  
A: 减小 `MIN_CLAP_INTERVAL`（如改为 80ms）。

**Q: 蜂鸣器有声音但设备不动？**  
A: 检查继电器连接，确认 GPIO 26/27 有输出。可用万用表测量继电器 IN 引脚电压。

## 扩展开发建议

1. **增加更多命令**: 识别 4 次、5 次拍手，控制更多设备
2. **音调识别**: 使用 FFT 分析频率，区分拍手和其他声音
3. **在线语音识别**: 将音频通过 WiFi 发送到云端 API（如百度语音、阿里云）
4. **本地语音识别**: 使用 Edge Impulse 训练本地语音模型烧录到 ESP32

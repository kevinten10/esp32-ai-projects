# 手势识别控制器

基于 ESP32 + APDS-9960 传感器的手势控制系统，通过手掌滑动控制设备。

---

## 功能特性

| 功能 | 说明 |
|------|------|
| 手势识别 | 识别上/下/左/右四方向手势 |
| PWM 调光 | 向上/向下控制 LED 亮度（256级） |
| 设备控制 | 向左/向右切换灯光/风扇继电器 |
| 接近检测 | 检测手靠近传感器 |
| OLED 显示 | 显示当前手势、操作结果、设备状态 |
| 蜂鸣器反馈 | 不同手势有不同音调提示 |

---

## 手势命令

| 手势 | 动作 | 蜂鸣音调 |
|------|------|---------|
| 向上滑动 ↑ | PWM 亮度 +12% | 高音短鸣 |
| 向下滑动 ↓ | PWM 亮度 -12% | 低音短鸣 |
| 向左滑动 ← | 切换灯光开/关 | 中音短鸣 |
| 向右滑动 → | 切换风扇开/关 | 中音短鸣 |

## 引脚连接

```
ESP32            APDS-9960 传感器
GPIO 21 (SDA) -> SDA
GPIO 22 (SCL) -> SCL
GPIO 4        -> INT（中断，可选）
3.3V          -> VCC（注意：仅3.3V！不可接5V）
GND           -> GND

ESP32            OLED SSD1306
GPIO 21       -> SDA（与APDS-9960共用I2C总线）
GPIO 22       -> SCL
3.3V          -> VCC
GND           -> GND

ESP32            PWM LED
GPIO 13       -> 220Ω电阻 -> LED正极
GND           -> LED负极

ESP32            继电器模块
GPIO 26       -> IN1（灯光）
GPIO 27       -> IN2（风扇）
5V            -> VCC
GND           -> GND

ESP32            蜂鸣器
GPIO 25       -> 正极
GND           -> 负极
```

> ⚠️ **重要**: APDS-9960 工作电压为 **3.3V**，绝对不可接 5V，否则会损坏传感器！

## 快速开始

### 1. 安装依赖库

本项目使用 SparkFun APDS-9960 库，已在 `platformio.ini` 中配置：

```ini
lib_deps =
    sparkfun/SparkFun APDS9960 RGB and Gesture Sensor@^1.4.3
```

### 2. 编译上传

```bash
cd projects/gesture-control
pio run --target upload
```

### 3. 验证传感器

上传后打开串口监视器（115200），应看到：

```
=== ESP32 手势识别控制器 ===
OLED 初始化成功
APDS-9960 初始化... 成功
手势识别已启用
接近传感器已启用
系统就绪！
  向上滑动 -> 增加亮度
  向下滑动 -> 降低亮度
  向左滑动 -> 切换灯光
  向右滑动 -> 切换风扇
```

## 手势操作技巧

### 正确手势姿势

```
         ↑ 向上滑动
         |
← ----[APDS]---- →
向左    |         向右
         ↓ 向下滑动
```

1. 手掌距传感器 **5-15cm**
2. 匀速滑动，速度适中（不要太快或太慢）
3. 每次手势只做一个方向
4. 等待蜂鸣提示音再做下一个手势

### 常见手势失败原因

- 距离太近（< 5cm）或太远（> 20cm）
- 滑动速度太快或太慢
- 环境光线太强（遮住传感器玻璃）
- 传感器初始化失败（检查接线）

## 传感器原理

APDS-9960 通过以下方式检测手势：
1. 发出红外光（IR LED）
2. 用 4 个方向的光感器接收反射光
3. 分析各方向光强变化序列
4. SparkFun 库将序列转换为方向事件

## 常见问题

**Q: 串口显示 "APDS-9960 初始化失败"？**  
A: 
1. 检查接线（SDA→21, SCL→22）
2. 确认供电为 3.3V
3. 用 I2C 扫描程序检查 APDS-9960 地址（应为 0x39）
4. 检查 I2C 线是否有上拉电阻（大多数模块板载4.7kΩ上拉）

**Q: OLED 和 APDS-9960 I2C 冲突？**  
A: 两者 I2C 地址不同（OLED=0x3C, APDS=0x39），可共用总线，无冲突。

**Q: 手势识别不准确？**  
A: 适当降低环境光强度，遮住强光。APDS-9960 在强光下手势识别性能下降。

**Q: 蜂鸣器持续叫？**  
A: 如使用无源蜂鸣器，`tone()` 函数正常工作。如使用有源蜂鸣器，将代码中 `tone()` 改为 `digitalWrite(BUZZER_PIN, HIGH/LOW)`。

## 扩展开发建议

1. **增加颜色检测**: APDS-9960 支持 RGB 颜色感应，可读取物体颜色
2. **亮度自动调节**: 用环境光传感器（ALS）自动调节显示亮度
3. **更多手势命令**: 结合接近值区分"近距快速"和"远距慢速"手势
4. **WiFi 联动**: 将手势命令发送到智能家居服务器

# ESP32 433MHz RF 智能网关

将 ESP32 变成 433MHz 射频智能网关，无需改造即可接管市售射频智能插座、遥控开关、窗帘电机等设备，并接入 Home Assistant。

---

## 功能特性

| 功能 | 说明 |
|------|------|
| 学码 | 自动学习市售 433MHz 遥控器的编码 |
| 发射 | 发送 433MHz 信号控制插座/开关/窗帘 |
| 持久化 | 学到的设备保存到 Flash，重启不丢失 |
| Web 控制台 | 手机/电脑浏览器直接操控 |
| MQTT 接入 | 接入 Home Assistant，自动设备发现 |
| 按房间分组 | 支持按房间组织设备 |

## 引脚接线

```
ESP32              RF 发射模块 (FS1000A)
GPIO 17 ────────> DATA
5V      ────────> VCC（5V供电射程更远）
GND     ────────> GND
                  ANT ──> 17cm 导线（天线）

ESP32              RF 接收模块 (MX-RM-5V)
GPIO 16 <──────── DATA
5V      ────────> VCC
GND     ────────> GND
                  ANT ──> 17cm 导线（天线）

ESP32              OLED
GPIO 21 ────────> SDA
GPIO 22 ────────> SCL
3.3V    ────────> VCC
GND     ────────> GND
```

> ⚡ **天线重要性**: 必须在发射和接收模块上各连接 17cm 导线作为天线，否则有效距离不足 1 米。

## 快速开始

### 1. 修改 WiFi / MQTT 配置

编辑 `src/main.cpp`：

```cpp
const char* WIFI_SSID   = "你的WiFi名称";
const char* WIFI_PASS   = "你的WiFi密码";
const char* MQTT_SERVER = "192.168.1.20";   // HA/Mosquitto IP
```

### 2. 编译上传

```bash
cd projects/rf-gateway
pio run --target upload
```

### 3. 添加设备（学码）

1. 打开串口监视器（115200），查看 ESP32 IP
2. 浏览器访问 `http://<IP>`
3. 填写设备名称和房间
4. 点击**「学习「开」码」**，在 8 秒内按下遥控器的"开"按钮
5. 看到"学码成功"后，点击**「学习「关」码」**（如果开关码不同）
6. 点击**「保存设备」**
7. 设备出现在控制面板，即可点击控制

### 4. 验证控制

- 点击设备的"开"按钮，对应插座应该上电
- 如果无反应，尝试调整发射模块与插座距离

## MQTT 接口

```bash
# 开启第 0 个设备
mosquitto_pub -h 192.168.1.20 -t "home/esp32-rf/0/set" -m "ON"

# 关闭第 0 个设备
mosquitto_pub -h 192.168.1.20 -t "home/esp32-rf/0/set" -m "OFF"

# 查看第 0 个设备状态
mosquitto_sub -h 192.168.1.20 -t "home/esp32-rf/0/state"

# 手动发射指定 RF 码（用于测试）
curl "http://192.168.1.xxx/api/rawsend?code=16711680&protocol=1&bits=24"
```

### Home Assistant 自动发现

MQTT 连接后，ESP32 会自动向 Home Assistant 发布设备发现消息。HA 会自动创建 switch 实体，无需手动配置 `configuration.yaml`。

在 HA 中：**设置 → 设备与服务 → MQTT → 会看到 "ESP32 RF 网关"**

## 常见问题

**Q: 点击控制但插座没反应？**  
A:
1. 检查 RF 发射模块是否接 5V（接 3.3V 射程可能不足）
2. 天线是否连接（17cm 单芯线）
3. 用 Serial Plotter 或串口查看发射日志，确认编码正确
4. 靠近插座测试（1米内），排除距离问题

**Q: 学码超时，接收不到信号？**  
A:
1. RF 接收模块是否接 5V
2. 天线是否连接
3. 确认遥控器电池有电
4. 部分遥控器使用 315MHz，不兼容 433MHz 模块

**Q: 学到的码重启后消失？**  
A: 代码使用 ESP32 的 `Preferences`（NVS Flash）持久化存储，正常情况下重启不丢失。如果丢失，可能是 Flash 擦写失败，检查串口日志。

**Q: 控制窗帘（三个按钮：开/停/关）？**  
A: 分别学习三个按钮（开、停、关），保存为三个设备（如"客厅窗帘-开"、"客厅窗帘-停"、"客厅窗帘-关"）。

## 扩展开发建议

1. **场景控制**: 在 HA 中创建「回家」场景，一键开启所有灯和插座
2. **定时控制**: HA 自动化在每天固定时间开/关设备
3. **语音控制**: 通过 HA 接入小爱/天猫精灵，语音控制所有 RF 设备
4. **状态同步问题**: RF 是单向通信，ESP32 无法知道插座真实状态。可考虑在插座处加电流传感器（非侵入式 CT）反馈
